# 상권 단위 분석 · 통계적 신뢰지표 종합 보고서

> 스타벅스 입지 PU 모델의 ① **상권(commercial district) 단위 재구성** 과 ② **신뢰성 평가지표** 를 정의부터 결과·해석까지 정리.
> 기준 데이터: 22,305 카페(스벅 681 / 비스벅 21,624), base rate = 3.05%. 검증 = spatial GroupKFold(5) OOF.

# 제 1 부. 상권(Commercial District) 단위 분석

## 1.1 동기 — 왜 좌표가 아니라 상권인가

- **PU 라벨 타당성(팀 논의)**: 스벅이 *이미 있는 상권* 안의 비스벅 카페는 "스벅이 못 가는 곳"이 아니라 "이미 스벅이 점유한 상권의 일부" → 음성(U)으로 보기 부적절.
- 그러나 이를 **"그 카페들을 제외"** 로 구현하면 함정: 가장 애매한(=스벅과 닮은) 카페를 빼므로 평가만 쉬워져 성능이 *허수로* 오른다.
- **올바른 해법 = 분석 단위를 카페 → 상권으로 격상.** 카페를 지우지 않고, 자연스러운 비즈니스 단위(상권)에서 "이 상권이 스벅을 유치할 상권인가"를 묻는다.

### 검증: 카페 제외 방식의 함정 (정직한 비교)

| 설정 | 음성(U) 정의 | 평가셋 | PU-AUC | 판정 |
|---|---|---|---|---|
| baseline | 전체 비스벅 | 전체 | 0.677 | 기준 |
| B1 (clean-U) | 비스벅상권 카페만(학습) | **전체** | **0.614** | 공정평가 시 *하락* |
| B2 (clean-eval) | 비스벅상권 카페만 | 스벅상권 제외 | **0.893** | **허수(평가 축소)** |

- 스벅상권 내 비스벅 카페 = 전체 U의 **75%(16,135개)**. 이들이 가장 정보량 많은 hard-negative이자 가장 애매한 점.
- **B1**: 이들을 학습 음성에서 빼면(전체 공정평가) 외곽 카페만 보고 경계를 못 배워 *하락*(0.61).
- **B2**: 평가에서도 빼면 문제가 쉬워져 *허수 상승*(0.89, lift 8배). → "카페 제외"는 답이 아님.
- **결론**: 단위를 상권으로 올리는 방식(아래)이 정답.

## 1.2 상권 정의 — DBSCAN auto-eps

### 원리: k-거리 elbow로 eps 자동 결정 (클러스터링 단계와 동일 방법론)
1. 각 카페의 **k번째(=min_samples=5) 최근접 카페 거리** 계산
2. 22,305개 거리를 **내림차순 정렬** → 곡선: 좌측=고립점(큼), 우측=밀집점(작음)
3. **무릎점(2차 미분 최대 = 곡률 급변)** = 노이즈↔군집 전이 지점 → 그 거리를 eps로
4. 결과 **eps = 223m**, min_samples = 5 (haversine 거리)

> 임의값(150m)이 아니라 *데이터가 정한* 223m. "서울 카페 밀도가 자연히 갈리는 지점."

### 노이즈 처리
- eps·ms 미달 고립 카페 **831개(3.7%, 스벅 15 포함)** = 상권 미형성.
- **노이즈 최근접 흡수** 채택(고립 카페를 가장 가까운 상권에 편입) → 전 카페 배정(커버 100%). 흡수 유무로 AUC 0.791↔0.789(무차이)라, 데이터 손실 없는 흡수를 채택. 단 흡수된 고립 스벅 15개는 *특수입지*로 별도 언급(외곽·DT·랜드마크형; 클러스터링 DBSCAN 노이즈 14개와 일맥상통).

### 최종 상권 구조
- 상권 **170개** (스벅상권 87 / 비스벅상권 83), 상권 크기 중앙값 13 카페.
- **한계**: 거대상권 chaining — single-link 연결로 도심 상업벨트 일부가 한 덩어리(최대 6,095카페). HDBSCAN으로 분할 시도했으나 성능 저하(아래).

## 1.3 상권 정의 고도화 비교 (robustness)

| 정의 | 상권수 | 카페커버 | 상권 PU-AUC |
|---|---|---|---|
| **DBSCAN auto-eps(223m)** | 170 | 0.96 | **0.791** (최적·최정당) |
| DBSCAN 250m | 134 | 0.97 | 0.776 |
| DBSCAN 150m | 488 | 0.89 | 0.760 |
| DBSCAN 100m | 905 | 0.71 | 0.726 |
| HDBSCAN mcs10 | 420 | 0.63 | 0.659 |
| HDBSCAN mcs20 | 133 | 0.68 | 0.620 |

- **거칠수록(상권 클수록) AUC↑** (집계 노이즈 상쇄). auto-eps가 균형점.
- 합리적 정의(DBSCAN 100~250m, auto-eps) 전반에서 **0.73~0.79로 일관** → 상권단위 우월성은 *정의 방식에 robust*(파라미터 운빨 아님).
- **HDBSCAN은 오히려 저하**(가변밀도가 여기선 상권을 잘게 쪼개 커버↓) → *정석 도구가 항상 낫진 않음*, DBSCAN auto-eps 채택 근거.

## 1.4 집계·라벨 정의
- **상권 피처** = 구성 카페 15개 피처의 **평균**(개별 카페 노이즈 상쇄).
- **상권 라벨** = 구성 카페 중 스벅이 1개라도 있으면 양성(스벅상권), 없으면 U.
- 피처가 전부 *입지 환경*(주변 교통·상권밀도)이라 "스벅 유무"가 피처에 직접 새지 않음(누수 아님).

## 1.5 상권 단위 PU 결과
- **상권 PU-AUC(OOF, spatial CV) = 0.789**, bootstrap **95% CI [0.720, 0.852]**.
- 카페단위 0.677(CI [0.659,0.698]) 대비 **공정·robust한 실질 개선** (단위 격상의 효과; 카페 좌표·지오코딩 노이즈가 상권 평균으로 안정화).

## 1.6 서울대 캠퍼스를 '하나의 상권'으로 채점
- 교내 15개 카페 평균 → **상권 적합도 0.31, 상권 중 16백분위(최하위권)**.

# 제 2 부. 통계적 신뢰지표

## 2.0 왜 다지표인가
- **불균형(1:31.8)**: 정확도 97%는 "전부 비스벅"으로 달성 → 무의미.
- **PU 구조**: 진짜 음성 부재 → precision/recall/F1/accuracy 편향(U 속 숨은 양성을 오답 처벌).
- → **① 순위지표 ② 운영점 ③ 안정성 ④ 외적타당 ⑤ 가정정합** 다층으로, 각 지표가 PU-AUC의 사각을 보완.

각 지표: **정의 · 원리 · 통계적 보완 · PU-AUC 대비 이득 · 결과 · 해석 · 한계**

## A. 순위 성능

### A1. PU-AUC (주력)
- **정의**: `roc_auc_score(스벅라벨, score)` — U를 음성처럼 둔 ROC 면적.
- **원리**: P(무작위 스벅 score > 무작위 비스벅 score). 임계값 무관.
- **결과**: 카페 0.678 / **상권 0.789**.
- **해석**: "실제 스벅을 카페 군중 위로 띄울 확률." PU라 절대값은 과소(진짜 AUC의 단조변환) → *상대비교*엔 일관.
- **한계**: 운영점·불확실성·확률의미·모델의존을 못 봄 → 이하 보완.

### A2. AP (PR곡선 면적)
- **정의**: Σ(Rₙ−Rₙ₋₁)Pₙ. **원리**: 상위권 양성 밀도에 민감.
- **PU-AUC 대비 이득**: ROC는 거대 음성에 낙관적 → **불균형에서 AP가 현실적**.
- **결과**: 0.058 (base 0.031의 ~1.9배). **한계**: PU 하한.

### A3. Lift@k / Lift 곡선
- **정의**: precision@top-k / base_rate. **원리**: 상위 k% 양성 농축 배수.
- **이득**: AUC가 못 주는 **의사결정 예산(상위 k%)별 가치**.
- **결과**: **lift@5% = 2.88**(정점), 20%=2.09, 50%=1.50.
- **해석**: "상위 5%만 봐도 스벅 밀도 평균의 2.9배." **한계**: PU 하한.

### A4. Precision@k
- **정의**: 상위 k 중 실제 양성 비율. **이득**: 실제 *결정 품질*(AUC는 임계무관).
- **결과**: top5% 0.088. **한계(중요)**: **PU 하한** — 숨은 양성 때문에 실제는 더 높음.

### A5. Recall@k / Cumulative Gains
- **정의**: TP@k / P (커버리지). **이득**: 예산-커버리지 trade-off.
- **결과**: top20%=41.7%, top50%=74.9%. **해석**: 진짜 라벨만 써 편향 없음.

### A6. Lee & Liu PU-score
- **정의**: recall² / Pr(예측=양성). **원리**: 진짜 음성 없이 F1 단조대용(precision ∝ recall·P(y)/P(predict)).
- **이득**: AUC(임계무관)가 못 하는 **임계값 선택·운영점 품질**을 진짜 음성 없이.
- **결과**: gate 1.05, 최대 **1.16 @ thr 0.38**. **한계**: 0~1 아님, 절대해석 난해.

## B. 통계적 안정성

### B1. fold AUC 평균±std
- **정의**: 공간 fold별 AUC 평균/표준편차. **이득**: 단일값의 *분산*. **결과**: 0.675±0.021(지역 무관 일관).

### B2. Bootstrap 95% CI
- **정의**: (s,score) 1,000회 복원추출 AUC의 2.5/97.5 백분위. **원리**: 표본변동 불확실성.
- **이득**: 점추정에 **유의성·정밀도** 부여. **결과**: 카페 [0.659,0.698] / 상권 [0.720,0.852] — 둘 다 0.5와 분리.
- **한계**: 단순 bootstrap은 공간상관 무시(B1과 함께 봐야).

### B3. 모델간 점수 순위일치 (Spearman)
- **정의**: GBM·RF·XGB·LogReg·SVM 점수 쌍별 Spearman. **원리**: 다른 알고리즘이 같은 입지를 같게 랭킹하면 결과가 모델 산물 아님.
- **이득**: "AUC 비슷"보다 강함 — *같은 카페* 지목을 보장.
- **결과**: 평균 **0.874**(최소 0.772) = 매우 강한 일치.

## C. 외적 타당성 (최강)

### C1. OOT(out-of-time) recall@top-k%
- **정의**: 데이터수집 *이후* 신규 입점 스벅 8곳을, 모델 상위 k% 카페가 인근(≤0.3km)에서 덮는 비율.
- **원리**: 학습에 없던 *미래의 진짜 양성*으로 검증 → in-sample CV가 못 주는 일반화.
- **이득**: PU-AUC는 현재분포 내 CV. OOT는 **시간 분리 정답** → 가장 설득력.
- **결과**: top5%=0.50, 10%=0.625, **20%=0.875**(8곳 중 7곳 덮음). **한계**: 좌표 근사·N=8.

## D. 가정·보정

### D1. Elkan c / 추정 class prior π
- **정의**: c=E[score|라벨양성]≈P(s=1|y=1), π̂=(P/c)/N. **원리(SCAR)**: 라벨양성 평균점수=c.
- **이득**: AUC가 못 주는 **절대 유병률·확률 보정**. **결과**: c=0.567, **π≈5.4%**. **한계**: SCAR(스벅 비랜덤 라벨) 근사.

### D2. PU 신뢰도(순위) 곡선 — 단조성 ρ
- **정의**: score 십분위 → 실제 라벨양성률, 단조성 Spearman ρ. **원리**: 진짜 음성 부재로 절대 calibration 부당 → 점수↑→스벅률↑ *단조성*만 확인.
- **이득**: AUC(쌍별)가 못 보는 **수준별 일관성** → 점수를 연속 적합도로 쓰는 정당성.
- **결과**: **ρ=0.976**(거의 완벽 단조). **한계**: 절대 calibration 아님(c배 눌림).

# 결론

1. **상권 단위 재구성**으로 팀의 통찰("스벅상권 카페는 U 아님")을 *부풀림 없이* 실현: 카페 제외(B1/B2)는 하락/허수, **단위 격상**이 정답. DBSCAN auto-eps(223m, 클러스터링과 동일 방법론)로 상권 정의 → **PU-AUC 0.677→0.789**(공정·robust), 서울대 캠퍼스 = **상권 16백분위 최하위**(타 대학 80~98).
2. **신뢰성**은 단일 AUC가 아닌 다지표로 입증: 순위(A)+안정성(B, CI·모델일치 0.874)+외적타당(C, OOT 87.5%)+가정정합(D, ρ 0.976). 그리고 **모든 절대 정밀도 지표는 PU 하한 = 실제는 더 좋다.**
3. 목적이 입지추천이므로 정확도가 아닌 **lift·gains·precision@k**가 적합 지표 — "상위 5% 2.9배, 상위 20%로 스벅 42% 회수."
