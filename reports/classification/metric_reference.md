## 0. 왜 단일 지표(특히 정확도)로는 안 되나

- **극단 불균형(1:31.8)**: "전부 비스벅" 예측만으로 정확도 97% → 정확도·오류율은 무의미.

# A. 순위(ranking) 성능 지표

## A1. PU-AUC (주력)
- **정의**: `roc_auc_score(is_starbucks, score)`. U를 음성처럼 둔 ROC 곡선 하부면적.
- **원리**: 무작위 양성 1개가 무작위 음성 1개보다 높은 점수를 받을 확률 = P(score⁺ > score⁻). 임계값과 무관(전 임계값 평균), 순위만 반영.
- **우리 결과**: **0.678** (fold 0.675±0.021).
- **해석**: "실제 스벅을 비스벅 카페 군중 위로 띄울 확률 67.8%." 0.5=무작위, 1=완전분리.
- **PU 특수성**: U의 숨은 양성이 음성으로 세어져 **절대값은 진짜 AUC의 과소추정**. 단 SCAR 가정하 **진짜 AUC의 단조변환** → 모델·방법 *상대비교*에는 일관(그래서 5전략·16모델 비교의 기준).
- **한계 / 무엇을 못 보나**: ⓐ 특정 운영점(상위 k%)의 품질을 모름 ⓑ 상위권 집중도(어디서 잘 맞나)를 모름 ⓒ 점수의 확률적 의미(보정) 모름 ⓓ 한 번의 점추정이라 불확실성 모름. → A2~A6, B, C, D로 보완.

## A2. AP (Average Precision, PR곡선 면적)
- **정의**: precision-recall 곡선 하부면적 = Σ (Rₙ−Rₙ₋₁)·Pₙ.
- **원리**: 모든 임계값에서 (precision, recall)을 적분. **양성을 상위에 모으는 능력**에 민감.
- **우리 결과**: **0.058** (base rate 0.031 대비 ~1.9배).
- **PU-AUC 대비 이득**: ROC-AUC는 거대한 음성 다수 덕에 **불균형에서 낙관적**으로 보일 수 있음(FPR 분모가 큼). **AP는 양성 검색에 집중**해 불균형에서 더 현실적. AUC가 "전체 순위"라면 AP는 "상위권 양성 밀도".
- **통계적 보완**: 불균형 상황에서 ROC의 과대낙관을 교정.
- **해석**: 절대값은 낮아 보이나(0.058), base 0.031의 약 1.9배 → 무작위보다 양성을 앞에 모음. (PU라 이 또한 하한)
- **한계**: 운영점 비특정. PU 하한.

## A3. Lift 곡선 / lift@k
- **정의**: `lift@k = precision@top-k / base_rate`. (base = P/N = 0.0305)
- **원리**: "상위 k를 골랐을 때 무작위 대비 양성 밀도가 몇 배". 마케팅/타겟팅의 표준 gains 분석.
- **우리 결과**: **lift@5% = 2.88(정점)**, 10%=2.38, 20%=2.09, 50%=1.50 → 100%에서 1.0 수렴.
- **PU-AUC 대비 이득**: AUC는 *전역 순위*만 → "상위 5%만 보면 얼마나 이득?"에 답 못함. **lift는 의사결정 예산(상위 k%)에 직접 대응** → 입지추천 우선순위의 실질 가치를 수치화.
- **해석**: "모델 상위 5% 후보만 봐도 스벅형 밀도가 평균의 2.9배." 정점이 상위 5%라는 건 *최상위에 신호가 집중*됨을 의미.
- **한계**: PU 하한(숨은 양성 미포함 → 실제 lift는 더 큼). 정점 위치는 표본 변동.

## A4. Precision@k
- **정의**: 상위 k개 중 실제 양성 비율 = TP@k / k.
- **원리**: "k개를 찍으면 몇 개가 진짜 스벅?" 고정 예산 정밀도.
- **우리 결과**: top5% 0.088, top20% 0.064, top681(=#P) ≈ 0.08.
- **PU-AUC 대비 이득**: AUC는 임계 무관 → 실제 *결정 품질*을 못 줌. precision@k는 운영점 결정 품질.
- **해석**: 상위 5%(1,115곳)의 8.8%가 실제 스벅(base 3.1% → 2.9배).
- **한계(중요)**: **PU 하한** — U 속 미입점 유력지가 "오답"으로 계산되므로 **실제 precision은 더 높다.** 발표 시 "보수적 하한" 명시.

## A5. Recall@k / Cumulative Gains
- **정의**: `recall@k = TP@k / P` (전체 양성 중 상위 k가 회수한 비율). 누적 gains 곡선.
- **원리**: 커버리지 관점 — "상위 k%로 전체 스벅의 몇 %를 잡나".
- **우리 결과**: top10%=23.8%, **top20%=41.7%**, top30%=54.8%, **top50%=74.9%**.
- **PU-AUC 대비 이득**: precision의 짝 지표. AUC 대신 **예산-커버리지 trade-off**를 직관적으로 보여줌(상위 20%만 봐도 스벅 42% 발견).
- **해석**: 진짜 라벨만 쓰므로 **편향 없음**(분모 P가 확정). gains 곡선이 무작위 대각선 위 = 모델 유효.
- **한계**: precision과 trade-off(k 키우면 recall↑ precision↓).

## A6. Lee & Liu PU-score
- **정의**: `PU-score = recall² / Pr(예측=양성)`. recall은 라벨양성에서, Pr(예측=양성)은 전체에서.
- **원리(왜 이 형태)**: 진짜 음성이 없으면 precision을 직접 못 구한다. 그런데 F1 ∝ recall·precision이고 precision = recall·P(y=1)/P(predict=1) 이므로, **F1의 단조 대용으로 r²/P(predict=1)** 을 쓰면 **진짜 음성 없이 추정 가능**(Lee & Liu 2003). 임계값 선택·모델 비교용 PU 전용 점수.
- **우리 결과**: gate(0.534)에서 1.05, **최대 1.16 @ thr 0.383**.
- **PU-AUC 대비 이득**: AUC는 임계 무관이라 *운영점 품질·임계값 선택*에 못 씀. PU-score는 **진짜 음성 없이도 F1류 품질을 임계값별로 비교** → 게이트 임계 선택의 정당화.
- **해석**: 0~1 아님(비율). 값 자체보다 *임계값에 따른 곡선의 형태/최댓값 위치*가 정보. 최대가 thr 0.38 부근 = 그 근처가 recall·정밀 균형 최적.
- **한계**: Pr(예측=양성) 추정 분산, 절대 해석 어려움.

# B. 통계적 안정성 (우연·특정모델 아님)

## B1. fold AUC 평균 ± 표준편차
- **정의**: 공간 5-fold 각 test fold 내부 AUC의 평균과 표준편차.
- **원리**: pooled OOF AUC는 fold 간 점수 스케일 차이에 민감 → fold별 산출 후 평균이 더 엄밀. std는 일반화 분산.
- **우리 결과**: **0.675 ± 0.021**.
- **PU-AUC 대비 이득**: 단일 pooled 값의 *분산/안정성*을 드러냄. std 작음 = fold(지역)마다 일관.
- **해석**: ±0.02 → 지역을 바꿔도 성능이 크게 안 흔들림.

## B2. Bootstrap 95% CI (PU-AUC)
- **정의**: (s, score) 쌍을 1,000회 복원추출 → 매번 AUC → 2.5/97.5 백분위.
- **원리**: 표본 변동에 따른 AUC 불확실성을 비모수적으로 추정.
- **우리 결과**: AUC 0.678, **95% CI [0.659, 0.698]**.
- **PU-AUC 대비 이득**: 점추정 0.678에 **유의성·정밀도**를 부여. 구간이 0.5와 명확히 분리 → "무작위 아님"을 통계적으로 입증.
- **해석**: 구간폭 ~0.04로 좁음 = 추정 안정. 하한 0.659도 무작위보다 확실히 높음.
- **한계**: 공간 자기상관 무시한 단순 bootstrap(다소 낙관) → fold std(B1)와 함께 봐야.

## B3. 모델간 점수 순위일치 (Spearman)
- **정의**: GBM·RF·XGB·LogReg·SVM의 카페별 OOF 점수 간 쌍별 Spearman 상관.
- **원리**: 서로 다른 알고리즘이 *같은 카페를 같게 랭킹*하면, 결론이 특정 모델의 산물이 아님.
- **우리 결과**: **평균 0.874, 최소 0.772**.
- **PU-AUC 대비 이득**: "여러 모델 AUC가 비슷"한 것보다 강한 증거 — AUC가 같아도 *다른 카페*를 맞힐 수 있는데, 순위일치는 **같은 입지를 지목**함을 보장. 결과의 모델-비의존성(robustness).
- **해석**: 0.87 = 매우 강한 일치 → 입지 순위는 알고리즘 선택에 둔감 = 신뢰.
- **한계**: 트리계열끼리 더 높을 수 있음(상관의 구조적 유사).

# C. 외적 타당성 (미지의 진실에서 작동) — *가장 강력*

## C1. OOT(out-of-time) 신규 스벅 recall@top-k%
- **정의**: 데이터 수집 *이후* 신규 입점 스벅 8곳에 대해, 그 지점 인근(≤0.3km) 카페 중 모델 상위 k% 안에 드는 곳이 있으면 "덮음". recall = 덮은 매장 비율.
- **원리**: 학습에 전혀 안 쓰인 *미래의 진짜 양성*으로 검증 → in-sample CV가 못 주는 진짜 일반화.
- **우리 결과**: top5%=0.50, 10%=0.625, **20%=0.875**, 50%=0.875 (8곳 중 7곳을 상위 20%가 덮음; 미적중=송파풍납 주거지).
- **PU-AUC 대비 이득**: PU-AUC는 *현재 분포 내* CV 평가. OOT는 **시간적으로 분리된 새 정답**으로 검증 → 발표에서 가장 설득력 있는 신뢰 근거.
- **해석**: 모델 고적합 지역이 실제 신규 입점을 선행 포착. 미적중 1곳은 "스벅의 주거지 확장"이라는 해석/한계.
- **한계**: 신규 매장 좌표 근사 + 인근 proxy + N=8 소표본 → 정밀 검증 아님(경향 확인).

# D. 가정·보정 (PU 전제 정합성)

## D1. Elkan c / 추정 class prior π
- **정의**: c = E[score | 라벨양성] ≈ P(s=1 | y=1)(라벨링 빈도). 추정 π = (P/c)/N.
- **원리(SCAR)**: 양성이 무작위로 라벨된다면 라벨양성의 평균 점수 = c. 진짜 양성 수 ≈ 라벨양성/c.
- **우리 결과**: **c = 0.567**, 추정 진짜 양성비율 **π ≈ 5.4%**(라벨 3.05% → 보정 시 ~5.4%).
- **PU-AUC 대비 이득**: AUC는 *절대 유병률·확률 의미*를 못 줌. c·π는 점수를 확률로 보정(score_cal)하고 "전체 카페 중 5.4%가 스벅형"이라는 절대 해석을 가능케 함.
- **해석**: U의 약 (5.4%−3.05%) 만큼이 "미입점 유력 숨은 양성" 규모와 정합.
- **한계**: SCAR 가정 의존(스벅은 비랜덤 라벨이라 근사). 단일 추정 → 여러 추정기 교차검증이 이상적.

## D2. PU 신뢰도(순위) 곡선 — 단조성 ρ
- **정의**: score 십분위(1~10) → 각 구간의 실제 라벨양성(스벅)률. 십분위-양성률 Spearman ρ.
- **원리**: 진짜 음성 부재로 절대 calibration(대각선 일치)은 부당. 대신 **점수↑ → 실제 스벅률↑** 의 *단조성*이 "점수 순위가 진짜 의미를 가짐"을 보증.
- **우리 결과**: **ρ = 0.976** (거의 완벽한 단조 증가). 상위 십분위 스벅률이 base(3.05%)의 수 배.
- **PU-AUC 대비 이득**: AUC는 쌍별 순위만 요약. 신뢰도곡선은 **점수 수준별로** 양성률이 일관 상승함을 보여 → 점수를 *연속 적합도*로 쓰는 정당성(Opt1·서울대 백분위 해석)을 뒷받침.
- **해석**: 적합도 점수를 구간으로 잘라 의사결정해도 단조적으로 신뢰 가능.
- **한계(중요)**: 절대 라벨양성률은 진짜 음성 부재로 c배 눌림 → *절대 calibration 아님*, 순위 신뢰도 확인용.

# 핵심 메시지 (발표 #5)

1. **목적이 입지추천**이므로 정확도가 아니라 **상위 k 집중도(lift)·정밀(precision@k)·커버리지(gains)** 가 적합 지표.
2. **PU 구조**상 모든 절대 정밀도 지표는 **보수적 하한**(숨은 양성) — 실제 성능은 더 좋다.
3. PU-AUC 하나로 못 보는 ⓐ운영점(A3~A6) ⓑ불확실성(B2) ⓒ모델-비의존(B3) ⓓ미래일반화(C1) ⓔ확률의미(D)를 **다지표로 메워 신뢰성을 입체적으로 입증**.
4. 결정적 신뢰 근거 = **OOT(상위 20%가 신규 스벅 87.5% 포착)** + **모델간 순위일치 0.874** + **신뢰도곡선 단조 0.976**.
